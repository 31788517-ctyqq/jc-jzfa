# 竞彩足球专家推荐趋势监控系统 - 项目需求文档

## 1. 项目概述

### 1.1 项目背景
对米斗数据平台（https://midou310.com）上每日竞彩足球比赛的专家推荐数据进行定时抓取、统计分析和趋势可视化展示，帮助用户直观了解每场比赛的专家推荐方向变化趋势。

### 1.2 项目目标
1. 定时抓取每日竞彩足球比赛列表
2. 对每场比赛的专家推荐数据（推荐方向 + 专家数量）进行 30 分钟间隔的持续采集
3. 以移动端友好的图表形式展现推荐趋势变化
4. 从每天上午 11:00 开始启动抓取，持续跟踪数据变化

| 项目 | 内容 |
|------|------|
| URL | `GET /mdsj/score/getExpertRecommData.do` |
| 参数 | `dataId={matchId}`、`type=0` |
| 返回 | `{code:1, data:[...]}` |
| 数据结构 | 数组中每条包含：`{type, num, data}` |

| num | 该推荐方向的专家数量 |
| data[] | 专家详情（avatar, nickName, expertId） |

## 3. 功能需求

### 3.1 定时抓取系统
| 功能 | 说明 |
|------|------|
| 启动时间 | 每天 **11:00 AM** 自动启动 |
| 抓取频率 | 每 **30 分钟** 抓取一次 |
| 结束时间 | 比赛结束后停止抓取（matchStatus >= 2） |
| 抓取流程 | ①登录获取token → ②获取当日比赛列表 → ③遍历每场比赛获取推荐详情 → ④存储数据 |

### 3.2 数据存储
| 字段 | 类型 | 说明 |
|------|------|------|
| match_id | string | 比赛ID |
| home_team | string | 主队名称 |
| away_team | string | 客队名称 |
| league_name | string | 联赛名称 |
| match_time | datetime | 开赛时间 |
| capture_time | datetime | 抓取时间戳 |
| recommend_direction | string | 推荐方向（胜/平/负/胜平/平负/胜负等） |
| expert_count | int | 该方向专家数量 |
| raw_data | json | 原始完整数据 |

### 3.3 数据展示页面（移动端）
| 功能 | 说明 |
|------|------|
| 比赛列表 | 展示当日所有竞彩比赛，显示主客队、联赛、时间 |
| 比赛详情 | 点击进入后展示该场比赛的推荐趋势图表 |
| 趋势图表 | 以 **折线图/堆叠面积图** 展示各推荐方向专家数量随时间的变化 |
| 图表要求 | 直观、可横向滑动查看完整时间轴、每条数据保留 |

### 3.4 图表设计（核心功能）
| 类型 | 说明 |
|------|------|
| 图表类型 | 堆叠面积图 或 多线折线图 |
| X轴 | 时间（从 11:00 开始，每 30 分钟一个刻度） |
| Y轴 | 专家数量 |
| 系列 | 每个推荐方向（胜、平、负、胜平、平负等）为一个系列 |
| 交互 | 支持触摸滑动查看各时间点数据、图例切换显示/隐藏 |
| 时间范围 | 从第一次抓取到比赛结束后的最后一次抓取 |

## 9. 比赛推荐命中标记功能

### 9.1 数据来源
对已完成的比赛，推荐 API `/mdsj/score/getExpertRecommData.do` 的返回数据中包含 `result` 字段：

| result值 | 含义 | 显示样式 |
|----------|------|----------|
| `1` | **命中** | 红色/绿色标记，显示"中"或"命中" |
| `0` | **未中** | 灰色标记，显示"未中" |
| 其他/null | **待开** | 显示"待开"或"等待中" |

### 9.2 API返回示例（命中标记）
```json
[
  {"type":"胜", "num":6, "result":1, "data":[...]},   // 胜方向 → 命中
  {"type":"平", "num":2, "result":0, "data":[...]},   // 平方向 → 未中
  {"type":"负", "num":1, "result":0, "data":[...]},   // 负方向 → 未中
  {"type":"胜平", "num":1, "result":1, "data":[...]}  // 胜平方向 → 命中
]
```

### 9.3 命中标记显示功能

#### 9.3.1 比赛列表页
- 在比赛列表的每场比赛卡片上，显示推荐结果的**汇总标记**
- **已完成+有推荐**的比赛，显示"中"/"未中"标签
- 标签样式：绿色底+白色字的"中"、灰色底+白色字的"未中"
- 点击标签可跳转到该场比赛详情查看详细命中情况

#### 9.3.2 比赛详情页 — 推荐方向命中标记
- 在比赛详情页的趋势图表下方，展示每个推荐方向的命中状态
- 每个推荐方向卡片包含：
  ```
  ┌─────────────────────────────┐
  │  胜                      ✅  │  ← 命中标记
  │  5位专家       已命中       │
  └─────────────────────────────┘
  ┌─────────────────────────────┐
  │  平                      ❌  │  ← 未中标记

  └─────────────────────────────┘
  ```
- 命中标记颜色：命中=绿色(#4CAF50)，未中=红色(#F44336)，待开=灰色(#9E9E9E)

#### 9.3.3 趋势图表中的命中标记
- 在堆叠面积图/折线图中，命中的推荐方向序列**高亮显示**（加粗或特殊颜色）
- 在图表最后一个时间点标注最终的命中结果（中/未中标记）
- 图表图例中，命中的方向名称旁显示"✓"标记

## 10. 推荐方向命中率统计

### 10.1 统计范围
| 项目 | 说明 |
|------|------|
| 时间范围 | 近 **30 天**（从当前日期向前推算） |
| 比赛类型 | 所有 **已完成的竞彩足球比赛**（matchStatus >= 2） |
| 推荐方向 | 所有推荐维度（胜、平、负、让球、总进球、半全场等） |
| 统计粒度 | 按方向汇总 + 按天趋势 |

### 10.2 统计指标

| 指标 | 计算公式 | 说明 |
|------|---------|------|
| 总推荐数 | count(该方向所有推荐) | 某推荐方向在所有已完成比赛中的总出现次数 |
| 命中数 | count(result == 1) | 该推荐方向命中的次数 |
| 未中数 | count(result == 0) | 该推荐方向未中的次数 |
| 命中率 | 命中数 / 总推荐数 × 100% | 关键指标 |
| 每日命中率 | 每日命中数 / 每日推荐数 × 100% | 按天展示趋势 |

### 10.3 数据获取方式
```
循环 30 天：
  1. 调用 /mdsj/score/footballDataList.do?time={dayTimestamp}
  2. 筛选 matchStatus >= 2（已完成的比赛）
  3. 对每场已完成比赛调用 /mdsj/score/getExpertRecommData.do?dataId={matchId}
  4. 提取每条推荐的 result 和 type 字段
  5. 汇总统计数据
```

### 12.2 核心框架说明

#### 12.2.1 微信原生小程序框架
- **框架**：微信官方提供的 MINA 框架
- **优势**：
  - 原生支持微信生态能力（登录、支付、分享、消息模板）
  - 开发者工具完善（调试、性能监控、真机预览）
  - 组件化开发（自定义组件）
  - WXS 脚本支持
- **文件结构**：`.wxml` + `.wxss` + `.js` + `.json`

#### 12.2.2 TDesign MiniProgram 组件库
- **来源**：腾讯 TDesign 团队开发，GitHub 开源
- **地址**：`npm i tdesign-miniprogram -S --production`
- **核心组件**：
  | 组件 | 用途 |
  |------|------|
  | `t-navbar` | 顶部导航栏 |
  | `t-cell` / `t-cell-group` | 比赛列表卡片 |
  | `t-tabs` | 页面标签切换（推荐/图表） |
  | `t-loading` | 加载骨架屏 |
  | `t-tag` | 状态标签（中/未中标记） |
  | `t-sticky` | 粘性布局 |
  | `t-pull-down-refresh` | 下拉刷新 |
  | `t-empty` | 空数据占位 |
  | `t-icon` | 图标库 |
- **基础库要求**：`^2.12.0`

#### 12.2.3 微信云开发 (CloudBase)
- **产品**：腾讯云开发平台（TCB），微信小程序原生集成
- **云函数**：
  - Node.js 运行环境，自动弹性伸缩
  - 无需管理服务器
  - 支持定时触发器（Cron 表达式）
  - 示例：
    ```javascript
    // 每30分钟触发一次的云函数定时器
    exports.main = async (event, context) => {
      // 1. 登录米斗数据获取token
      // 2. 获取比赛列表
      // 3. 遍历获取推荐详情
      // 4. 存储到云数据库
    };
    ```
- **云数据库**：MongoDB 文档型数据库，SDK内置
  - 集合划分：`matches`、`recommends`、`hit_rates`、`capture_logs`
  - 支持聚合查询（用于命中率统计）
- **云存储**：存储图片、日志等静态资源
- **HTTP API**：通过云函数暴露 HTTP 请求接口给小程序调用

#### 12.2.4 图表库 — ECharts 小程序适配
- **方案**：`ec-canvas` 组件（微信官方 ECharts 适配）
- **安装**：从 [echarts-for-weixin](https://github.com/ecomfe/echarts-for-weixin) 引入
- **支持的图表类型**：
  | 图表 | 用途 |
  |------|------|
  | 堆叠面积图 | 推荐方向专家数量变化趋势 |
  | 多线折线图 | 各方向命中率每日趋势 |
  | 横向柱状图 | 各方向命中率排名 |
  | 仪表盘 | 整体命中率概览 |

#### 12.2.5 定时任务方案
- **方案**：微信云开发云函数定时触发器
- **Cron 表达式**：
  - 抓取任务：`0 */30 11-23 * * *`（每天 11:00~23:59 每30分钟触发）
  - 命中率统计：`0 0 2 * * *`（每天凌晨2点重新计算命中率）
- **无服务器**：托管运行，无需维护服务器

## 13. 当天比赛推荐排名

### 13.1 功能概述
对当天所有竞彩比赛，根据各方向专家推荐数量进行排名，直观展示每场比赛哪个推荐方向专家最看好，以及该方向在全部比赛中的排位。

### 13.2 排名规则
| 维度 | 说明 |
|------|------|
| 排名依据 | 某场比赛中各推荐方向中**专家数量最大值** |
| 排位规则 | 按专家数量从高到低排序，数量相同则并列 |
| 排名范围 | 当天所有竞彩比赛 |
| 更新频率 | 每30分钟抓取后自动更新排名 |
| 显示字段 | 排名、主队 vs 客队、推荐方向、专家数量 |

### 13.3 排名计算示例

| 排名 | 比赛 | 最热方向 | 专家数 | 说明 |
|------|------|---------|--------|------|
| 1 | 拉赫蒂 vs 瓦萨 | **胜** | 6人 | 该场"胜"方向6位专家，为全场最高 |
| 2 | 赫尔辛基 vs 玛丽港 | **胜** | 5人 | 5位专家推荐"胜" |
| 3 | 洪卡 vs 埃尔维斯 | **平** | 4人 | 该场最热方向为"平" |

## 14. 按推荐方向筛选排名

### 14.1 功能概述

### 14.2 筛选规则
| 维度 | 说明 |
|------|------|
| 筛选方式 | 方向选择器（下拉/滑动选择器） |
| 可选方向 | 当天所有比赛中实际出现的推荐方向（胜、平、负、胜平、平负、让胜、让平、让负、总进球等） |
| 排序规则 | 按该方向专家数量从高到低 |
| 数据来源 | 最新一次抓取数据 |
