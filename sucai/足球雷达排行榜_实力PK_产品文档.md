## 一、产品定义与核心价值
### 1.1 产品定位
**实力PK**是足球雷达排行榜的核心模块，旨在通过**多维度量化算法**，将抽象的球队实力转化为标准化的**可视化数值体系**。系统支持**单场深度分析**与**多场横向PK**，帮助用户在复杂的赛事中快速识别优势方与价值洼地。

### 1.2 核心指标定义
系统以**主队视角**输出数据，`+`代表主队占优，`-`代表客队占优。

| 维度 | 定义 | 数据极性 | 算法来源 |
| :--- | :--- | :--- | :--- |
| **净胜球量化** | 基于攻防频次还原的真实预期净胜球。 | `+` 主队预期进球多 | 源自主客近期期望模块 |
| **胜平负交叉** | 基于真实历史赛果的交叉对冲，稳定性高于赔率。 | `+` 主队赛果占优 | 源自近期战绩统计 |
| **综合实力** | 结合静态基本面与动态近期状态的双轨评分。 | `+` 主队基本面强 | 源自足球全景智算分析系统 V6.0 |
| **攻守实力** | 基于功守道V24.0模型，细分进攻与防守的效率差。 | `+` 主队攻守平衡/压制 | 源自《功守道 · 八阵图 V24.0》 |

## 二、核心算法与数据字典
### 2.1 通用计算规则
1.  **占比计算**：
    *   **分母（列合计）**：始终为该列**绝对值之和**（用于防止正负抵消）。
    *   **单项占比**：$\frac{当前值}{列合计}$ （保留正负符号）。
2.  **总排序算法**：
    $$总排序 = \frac{净胜球占比 + 胜平负交叉占比 + 综合占比 + 攻守占比}{4}$$
3.  **动态PK逻辑**：
    *   当用户勾选多场点击“PK”时，**列合计**仅基于**当前勾选的行**重新计算，而非全量数据。

### 2.2 子维度算法详解
#### A. 净胜球量化 (Goal Difference Quantification)
*   **逻辑**：通过射门次数与效率还原底层攻防频次，计算预期进球差。
*   **第一阶段：还原底层攻防次数**：
    *   主队进攻次数 = $\frac{H_{rg}}{|H_{att}|}$
    *   主队被射次数 = $\frac{H_{rl}}{|H_{def}|}$
    *   客队进攻次数 = $\frac{A_{rg}}{|A_{att}|}$
    *   客队被射次数 = $\frac{A_{rl}}{|A_{def}|}$
*   **第二阶段：计算四维呼吸权重**：
    *   $W_{HA} = \frac{主队进攻次数}{主队进攻次数 + 客队被射次数}$
    *   $W_{AD} = \frac{客队被射次数}{主队进攻次数 + 客队被射次数}$
    *   $W_{AA} = \frac{客队进攻次数}{客队进攻次数 + 主队被射次数}$
    *   $W_{HD} = \frac{主队被射次数}{客队进攻次数 + 主队被射次数}$
*   **第三阶段：终极进球期望计算**：
    *   主队预期进球 ($\lambda_H$) = $(H_{vg} \times W_{HA}) + (A_{vl} \times W_{AD})$
    *   客队预期进球 ($\lambda_A$) = $(A_{vg} \times W_{AA}) + (H_{vl} \times W_{HD})$
*   **最终输出**：
    $$净胜球\_量化 = \lambda_H - \lambda_A$$

#### B. 胜平负交叉 (Result Cross Advantage)
*   **逻辑**：利用真实的历史赛果（胜平负场次）进行对冲，剔除赔率波动干扰，反映最真实的近期状态差异。
*   **公式**：
    $$CrossAdv = (H_{w\_cnt} + A_{l\_cnt}) - (H_{l\_cnt} + A_{w\_cnt})$$
    *   $H_{w\_cnt}$: 主队近10场胜场数
    *   $A_{l\_cnt}$: 客队近10场负场数
    *   $H_{l\_cnt}$: 主队近10场负场数
    *   $A_{w\_cnt}$: 客队近10场胜场数
*   **解读**：数值越高，说明主队拿分能力越强或客队失分越多。

#### C. 综合实力量化 (Composite Strength - Dual Track)
*   **逻辑**：70%静态硬实力 + 30%动态状态。
*   **公式**：
    $$TotalDiff = StaticDiff \times 0.7 + DynamicDiff \times 0.3$$
    *   **静态**：$StaticDiff = \frac{P_{home} - P_{away}}{P_{home} + P_{away}}$ （JS: `homePower`, `guestPower`）
    *   **动态**：$DynamicDiff = \frac{(WinPan_{home} + M_H) - (WinPan_{away} + M_A)}{(WinPan_{home} + M_H) + (WinPan_{away} + M_A)}$

#### D. 攻守实力量化 (Attack-Defense Dao V24.0)
*   **逻辑**：严格在后台按顺序执行以下数学模型计算，不允许跳过任何步骤，浮点数保留 4 位小数。
*   **公式**：
    $$A = (Adv_{攻} \times W_{攻}) + (Adv_{守} \times W_{守})$$

##### 第一阶段：进攻相对优势度 ($Adv_{攻}$)
1.  **赢球格局得分对冲**：
    $$S\_win\_H = (H_{w2} \times 3.6) + (H_{w1} \times 3.3) + (H_{d} \times 1.0)$$
    $$S\_win\_A = (A_{w2} \times 3.6) + (A_{w1} \times 3.3) + (A_{d} \times 1.0)$$
    $$R\_win = \frac{S\_win\_H - S\_win\_A}{S\_win\_H + S\_win\_A}$$
2.  **攻击力纯能效对冲**：
    $$P\_att\_H = \frac{H_{vg} \times H_{rg}}{H_{att}} \quad , \quad P\_att\_A = \frac{A_{vg} \times A_{rg}}{A_{att}}$$
    $$R\_att = \frac{P\_att\_H - P\_att\_A}{P\_att\_H + P\_att\_A}$$
3.  **进球厚度分布对冲**：
    $$D\_score\_H = (H_{s0} \times 0) + (H_{s1} \times 1) + (H_{s2} \times 2)$$
    $$D\_score\_A = (A_{s0} \times 0) + (A_{s1} \times 1) + (A_{s2} \times 2)$$
    $$R\_score = \frac{D\_score\_H - D\_score\_A}{D\_score\_H + D\_score\_A}$$
4.  **合成优势**：
    $$Adv_{攻} = (R\_win \times 0.4) + (R\_att \times 0.4) + (R\_score \times 0.2)$$

##### 第二阶段：防守相对优势度 ($Adv_{守}$)
1.  **输球空间与容错对冲**：
    $$S\_lose\_H = (H_{l1} \times 3.3) + (H_{l2} \times 3.6) + (H_{d} \times 1.0)$$
    $$S\_lose\_A = (A_{l1} \times 3.3) + (A_{l2} \times 3.6) + (A_{d} \times 1.0)$$
    $$R\_lose = \frac{S\_lose\_A - S\_lose\_H}{S\_lose\_A + S\_lose\_H}$$
2.  **防御纯能效对冲**：
    $$P\_def\_H = \frac{H_{vl} \times H_{rl}}{H_{def}} \quad , \quad P\_def\_A = \frac{A_{vl} \times A_{rl}}{A_{def}}$$
    $$R\_def = \frac{P\_def\_A - P\_def\_H}{P\_def\_A + P\_def\_H}$$
3.  **失球厚度与零封率对冲**：
    $$D\_leak\_H = (H_{k0} \times 2) + (H_{k1} \times 1) + (H_{k2} \times 0)$$
    $$D\_leak\_A = (A_{k0} \times 2) + (A_{k1} \times 1) + (A_{k2} \times 0)$$
    $$R\_leak = \frac{D\_leak\_H - D\_leak\_A}{D\_leak\_H + D\_leak\_A}$$
4.  **合成优势**：
    $$Adv_{守} = (R\_lose \times 0.4) + (R\_def \times 0.4) + (R\_leak \times 0.2)$$

##### 第三阶段：格局划分与权重融合
1.  **进攻权重**：
    $$W_{攻} = \frac{D\_score\_H + D\_score\_A}{D\_score\_H + D\_score\_A + D\_leak\_H + D\_leak\_A}$$
2.  **防守权重**：
    $$W_{守} = 1 - W_{攻}$$
3.  **最终输出**：
    $$A = (Adv_{攻} \times W_{攻}) + (Adv_{守} \times W_{守})$$

# 足球雷达排行榜·进球数预测 产品文档（V1.0 终极归档版）

---
## 一、产品定义与核心价值
### 1.1 产品定位
**进球数预测与PK模块**是足球雷达的**大小球研判核心工具**，旨在通过**多维度算法**输出标准化的进球预期，并支持**多场横向对比（PK）**。系统通过对比不同赛事的进攻预期与防守漏洞，辅助判断大小球走势。

### 1.2 核心指标定义
系统输出6个维度的标准化指标：

| 维度 | 定义 | 数据极性 | 算法来源 |
| :--- | :--- | :--- | :--- |
| **合计** | 各维度绝对值之和（用于计算占比）。 | `绝对值` | 通用计算规则 |
| **大球比例** | 综合主客及交锋的大球倾向。 | `越高` 大球概率越高 | 综合比例算法 |
| **攻防进球** | 基于射门次数与效率还原的物理进球预期。 | `绝对值` 预期进球数 | 射门还原法 (M3_A) |
| **实力进球** | 基于武力值与厚度的加权进球预期。 | `绝对值` 预期进球数 | 攻守权重法 (M3_B) |
| **交锋进球** | 基于双方历史交手的进球惯性。 | `绝对值` 预期进球数 | 交锋预测法 (M3.6) |
| **破甲和** | 衡量双方防线被击穿的难度总和。 | `越高` 防线越脆弱 | 绝对攻防战力 |

---
## 二、核心算法与数据字典
### 2.1 通用计算规则
1.  **数据缺失处理**：任何缺失的大球比例数据，默认取值 **50%**。
2.  **合计计算**：
    $$合计 = |大球比例| + |攻防进球| + |实力进球| + |交锋进球| + |破甲和|$$
3.  **占比计算**：
    $$单项占比 = \frac{当前值}{合计} \quad (保留正负符号)$$
4.  **动态PK逻辑**：
    *   当用户勾选多场点击“PK”时，**合计**与**占比**仅基于**当前勾选的行**重新计算。

### 2.2 子维度算法详解
#### A. 大球比例 (Big Ball Ratio)
*   **公式**：
    $$综合大球比例 = \frac{主队大球比例 + 客队大球比例 + 交锋大球比例}{3}$$

#### B. 攻防进球 (M3_A: Shot Restoration)
*   **公式**：
    $$攻防进球 = (主主进 \times W_{HA}) + (客客失 \times W_{AD}) + (客客进 \times W_{AA}) + (主主失 \times W_{HD})$$

#### C. 实力进球 (M3_B: Attack-Defense Weighting)
*   **公式**：
    $$实力进球 = (进攻期望 \times \Omega_{Atk\_H}) + (防守期望 \times \Omega_{Def\_H})$$

#### D. 交锋进球 (M3.6: Head-to-Head)
*   **公式**：
    $$交锋进球 = (G_6 \times 0.3) + (G_2 \times 0.7)$$

#### E. 破甲和 (Penetration Sum)
*   **公式**：
    $$破甲和 = \frac{客队防守力}{主队攻击力} + \frac{主队防守力}{客队攻击力}$$

## 一、产品定义与核心价值
### 1.1 产品定位
**热度PK**是足球雷达的**资金流向与市场情绪监测模块**。通过量化投注资金与欧指概率的偏离度，识别市场过热（诱盘）或冷藏（无庄）场次。数据**每半小时更新一次**。

### 1.2 核心指标定义
系统输出8个维度的标准化指标（主队视角）：

| 维度 | 定义 | 数据极性 | 应用场景 |
| :--- | :--- | :--- | :--- |
| **让球数 (rq)** | 亚洲让球盘口 | 数值 | 决定冷热算法分支 |
| **市场关注热度** | 市场总体关注量、成交量或活跃度指标 | `越高` 市场焦点 | 判断资金深度 |
| **冷热指数 ($R_{heat}$)** | 投注比例与理论概率的比值 | `>1` 热, `<1` 冷 | 核心判断依据 |
| **主队特征** | 主队基本面/盘口特征描述 | 文本标签 | 辅助定性分析 |
| **客队特征** | 客队基本面/盘口特征描述 | 文本标签 | 辅助定性分析 |
| **静态实力差** | 基本面硬实力差距 | 数值 | 验证热度合理性 |
| **亚指临盘** | 当前亚洲盘水位变化 | 数值 | 配合热度看赔付风险 |

---
## 二、核心算法与数据字典
### 2.1 通用计算规则
1.  **更新频率**：数据每 **30分钟** 自动刷新一次。
2.  **核心变量**：
    *   $hotFocusNum$: 市场关注热度 (`hotFocusNum`)
    *   $Bet_{home}$: 主队投注比例 (`homeBetPercent`)
    *   $Bet_{away}$: 客队投注比例 (`awayBetPercent`)
    *   $W$: 欧指临盘胜概率 (`lastWinRate`)
    *   $L$: 欧指临盘负概率 (`lastLoseRate`)
    *   $rq$: 让球数 (`rq`)

### 2.2 冷热指数算法 ($R_{heat}$)
系统根据 **让球数 (rq)** 自动切换计算公式与判定标准。

#### 场景 A：一球受让 (rq = -1)
*   **公式**：$R_{heat} = \frac{Bet_{home}}{W}$
*   **判定标准**：
    *   $R_{heat} > 1.20$ 🔥 **严重过热 (防冷)**
    *   $0.90 \le R_{heat} \le 1.10$ 🎯 **筹码均衡**
    *   $R_{heat} < 0.80$ 🧊 **逆向冷藏 (关注)**

#### 场景 B：一球让球 (rq = +1)
*   **公式**：$R_{heat} = \frac{Bet_{away}}{L}$
*   **判定标准**：
    *   $R_{heat} > 1.20$ 🔥 **严重过热 (防冷)**
    *   $0.90 \le R_{heat} \le 1.10$ 🎯 **筹码均衡**
    *   $R_{heat} < 0.80$ 🧊 **逆向冷藏 (关注)**

#### 场景 C：深盘受让 (rq <= -2)
*   **公式**：$R_{heat} = rqLosePercent \times lastRqLoseOdd$
*   **判定标准**：
    *   $R_{heat} \le 0.6$ 🔥 **过热 (让负难出)**
    *   $0.6 < R_{heat} \le 0.8$ ⚠️ **微热**
    *   $R_{heat} > 0.8$ 🎯 **正常**

#### 场景 D：深盘让球 (rq >= +2)
*   **公式**：$R_{heat} = rqWinPercent \times lastRqWinOdd$
*   **判定标准**：
    *   $R_{heat} \le 0.6$ 🔥 **过热 (让胜难出)**
    *   $0.6 < R_{heat} \le 0.8$ ⚠️ **微热**
    *   $R_{heat} > 0.8$ 🎯 **正常**
